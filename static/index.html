What is Deno?

Deno is an anagram of node. This very website was created with Deno! It was created by the same guy who created Node JS.

Why Deno?

The creator of Deno had some regrets about Node JS. Without getting too technical, his main concern was making sure Deno was more secure. For now, don't worry about learning Deno, as it's creator said "Node isn't going anywhere". However, if you are interested in what a more modern version of Node JS is, then Deno is it.

Deno vs. Node

By no means will Deno replace Node, for a long time. Deno is in it's first version, while Node has had plenty of updates and fixes through it's run over the years. However, Deno has the potential to replace Node if it receives widespread adoption.

Code in Node 
const request = require("request-promise")
const fetch = async () => {
  const req = await request.get({url: "https://google.com"})
  console.log(req.statusCode)
}              
const run = async () => {
    await fetch()
}
Code in Deno 
import * as request from "request-promise-repo-url"
const req = await request.get({url: "https://google.com"})
console.log(req.statusCode)

In the example above, we are trying to send a GET request to Google using an asynchronous function, which means we must await it. In Node JS, you can only use the await keyword in an async function. Since Deno reliieves us of that restriction, we don't need to wrap our await of the request.get() function, allowiing for quicker, and cleaner coding.

Pretty Cool!

Some things that are necessary (maybe even annoying) about Node are fixed in Deno. To keep it short, I'll summarize some examples:

  • Write typescript code without a tsconfig file
  • await can be ran anywhere, not just inside an async function
  • No need for package.json file or a node_modules folder, modules are decentralized and distributed by their creators.
  • more secure out of the box.

Try it out!

If you're on Mac or Linux, open terminal and run this command:

curl -fsSL https://deno.land/x/install/install.sh | sh

Windows:

iwr https://deno.land/x/install/install.ps1 -useb | iex

Then run:

deno run https://deno.land/std/examples/welcome.ts

You just ran your first Deno program! Learn more and start writing your own!
